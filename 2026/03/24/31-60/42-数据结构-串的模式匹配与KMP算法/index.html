【王道考研·数据结构】第四章 串：4.2 模式匹配、KMP算法与next/nextval数组

Austoin2026-03-242026-03-24

引言

第四章真正的硬核内容，其实不是“串的定义”，而是字符串模式匹配。

这一部分之所以重要，是因为它把前面的抽象串结构，真正落到了一个具体问题上：

在一个主串中，如何高效找到某个模式串第一次出现的位置？

最朴素的方法当然可以做，但效率不高；于是就有了考研重点中的重点：KMP 算法。

本篇会按下面这条主线完整展开：

  1. 什么是模式匹配
  2. 朴素模式匹配怎么做
  3. 为什么主串指针其实没必要回溯
  4. next 数组到底是什么意思，怎么手算
  5. nextval 为什么还能继续优化

PPT 中有大量动态图示，这里我会把它们背后的推导逻辑补全，让你不是“记住结论”，而是“真的会推”。

一、什么是字符串的模式匹配

给定：

  • 主串 S
  • 模式串 T

字符串模式匹配要解决的问题是：

在主串 S 中找到一个与模式串 T 值相同的子串，并返回它第一次出现的位置；如果不存在，则返回 0。

例如：

  • 主串：“嘿嘿嘿红红火火恍恍惚惚嗨皮开森猴开森笑出猪叫哈哈哈哈”
  • 模式串：“笑出猪叫”

若能找到，就返回该模式串在主串中的起始位置。

1. 模式串和子串不是一回事

这是 PPT 专门强调的一点。

  • 子串：一定是主串的一部分，所以一定存在于主串中
  • 模式串：只是你拿来做匹配的目标，不一定能在主串中找到

例如：

  • “笑出猪叫” 可能是主串的子串
  • “笑出喵叫” 可以作为模式串，但它未必是主串的子串

所以模式匹配不是“找一个子串”，而是“判断模式串是否在主串中出现”。

二、朴素模式匹配算法

1. 核心思想

设：

  • 主串长度为 n
  • 模式串长度为 m

主串中一共有 n - m + 1 个长度为 m 的候选子串。

朴素模式匹配（又叫 BF 算法）的思路非常直接：

  1. 取主串中第 1 个长度为 m 的子串与模式串比较
  2. 若不匹配，就比较下一个长度为 m 的子串
  3. 直到找到完全匹配的子串，或所有候选子串都试完

2. 指针版思路

教材更常用两个指针：

  • i：指向主串当前正在比较的字符
  • j：指向模式串当前正在比较的字符

若 S[i] == T[j]，则两者同时后移；
若失配，则：

这条式子的含义是：

  • 当前比较失败时，主串应回到“当前候选子串的下一个起点”
  • 模式串重新回到第 1 个字符开始匹配

3. 为什么是 i = i - j + 2

这是很多人死记硬背的地方，这里补一下推导。

假设失配前：

  • 当前候选子串起点是 k
  • 已经匹配到了模式串第 j 个字符

由于此前匹配了 j - 1 个字符，所以此时：

$$i = k + (j - 1)$$

于是原候选子串起点：

$$k = i - j + 1$$

下一个候选子串应从 k + 1 开始，因此新的主串位置：

$$i = k + 1 = i - j + 2$$

这就是这个公式的来源。

4. 代码实现

5. 时间复杂度

最坏情况下：每个候选子串都要比较 m 个字符，一共有 n - m + 1 个候选子串，因此：

$$O((n-m+1)m) = O(nm)$$

最典型的最坏情况是：

  • 主串：aaaaaaa...aaab
  • 模式串：aaaaab

前面每次都几乎匹配成功，只在最后一个字符失败，于是产生大量重复比较。

1. 朴素算法浪费在哪里

设模式串 T = "abaabc"。

说明在失配字符之前，主串当前这 5 个字符已经和模式串前 5 个字符完全相同。

这意味着：

  • 已经比较过的那部分信息不是“没用了”
  • 某些候选子串其实根本没必要再试

2. KMP 的核心直觉

主串指针不回溯，只有模式串指针回跳。

因为主串中已经比较过的字符信息还在：

  • 不匹配字符之前，主串前缀和模式串前缀一定相等
  • 真正需要做的是：找到模式串内部一个“还能继续接上”的位置

也就是：

让模式串向右滑动，使得“已经匹配成功的那一段”的某个前缀，重新和它的某个后缀对齐。

这正是 next 数组要记录的东西。

四、KMP 算法与 next 数组

1. next 数组的含义

在王道教材的 1 基下标写法里：

  • 当模式串第 j 个字符失配时
  • 模式串指针应跳到 next[j] 位置继续比较

特别地：

  • next[1] = 0

因为第 1 个字符就失配时，模式串已经没有可复用的信息了，只能比较主串中的下一个相邻子串。

2. next[j] 的本质

对于 j > 1，next[j] 可以理解为：

在 T[1...j-1] 中，最长相等“真前缀 = 真后缀”的长度，再加 1。

为什么要加 1？

因为我们记录的不是“长度”，而是下一次应该从模式串第几个字符继续比。

例如 T = "abaabc"：

  • j = 6 失配时，已经匹配的是 T[1...5] = "abaab"
  • 它的最长相等真前后缀是 "ab"，长度为 2
  • 所以下一步应从模式串第 3 个字符继续比较
  • 因此 next[6] = 3

五、next 数组怎么手算

这一节是 PPT 里图示最多、也是很多同学最容易“看懂动画但不会自己算”的地方。

1. 永远先写两项

对于王道这套写法：

  • next[1] = 0
  • next[2] = 1

原因：

  • 第 1 个字符失配，模式串无处可退，只能试下一个相邻子串
  • 第 2 个字符失配时，最多只能回到第 1 个字符重新匹配

2. 之后怎么推

求 next[j] 时，考察的是失配前已经匹配成功的部分 T[1...j-1]。

具体方法可以理解为：

  1. 在 j 前面画一条分界线
  2. 寻找 T[1...j-1] 的最长相等真前缀和真后缀
  3. 若这个公共部分长度为 k，则 next[j] = k + 1
  4. 若不存在，则 next[j] = 1

4. 一个实用记忆法

如果你考试时突然忘了公式，不要硬背，直接：

  • 先写出当前失配前已匹配成功的模式串前缀
  • 再找它的最长相等真前缀和真后缀
  • 公共部分长度 +1，就是 next[j]

这比死记结论更稳。

七、KMP 为什么主串指针不回溯

这是本章最核心的理解点，必须单独拎出来。

假设失配时主串指针在 S[i]，模式串指针在 T[j]，并且此前 T[1...j-1] 已匹配成功。

这意味着：

$$S[i-j+1…i-1] = T[1…j-1]$$

也就是说，主串这一段的内容我们已经完全知道。

所以接下来要做的不是“重新检查主串过去的字符”，而是：

  • 找到模式串内部可复用的前缀
  • 让它与刚才已经匹配成功的后缀对齐

因此：

  • 主串指针 i 不回退
  • 模式串指针 j 通过 next 或 nextval 回跳

这就是 KMP 与朴素算法的本质区别。

八、做题防坑指南

1. 先看下标体系

王道教材通常：

  • 串位序从 1 开始
  • next[1] = 0

如果换成 0 基数组写法，公式和代码会跟着变化，千万别混用。

2. 分清三个量

  • j：当前正在比较的模式串位置
  • next[j]：失配后模式串应该跳到的位置
  • nextval[j]：进一步跳过必然失配位置后的结果

3. next[j] 不是“公共前后缀长度”本身

在王道写法里，next[j] 记录的是下一次继续比较的模式串位置，它等于“最长相等真前后缀长度 + 1”。

九、经典考点小结

这一节真正会考的通常就下面几类：

  1. 解释模式匹配的定义：主串、模式串、返回位置
  2. 手推朴素模式匹配过程：尤其是失配后的回退公式
  3. 分析朴素算法复杂度：最坏 O(nm)
  4. 解释 KMP 的优化思想：为什么主串不回溯
  5. 手算 next 数组
  6. 手算 nextval 数组
  7. 写出或补全 KMP 代码
  8. 比较 BF、KMP、KMP 优化版的时间复杂度与无效比较次数

总结

把这一节压缩成一句话：

KMP 的本质，就是利用模式串自身的结构信息，避免主串回溯和重复比较。

你最后应该牢牢记住：

  1. 朴素匹配：失配后主串回到下一个候选子串起点，复杂度 O(nm)
  2. KMP：失配后主串不回溯，模式串按 next 回跳，总复杂度 O(m+n)
  3. next 数组：记录失配后模式串该跳到哪里继续比较
  4. nextval 数组：进一步跳过那些“跳过去也一定继续失配”的位置

如果你能自己手推出 abaabc、google、ababaa、aaaab 这些例子的 next / nextval，这章最难的部分就已经过关了。

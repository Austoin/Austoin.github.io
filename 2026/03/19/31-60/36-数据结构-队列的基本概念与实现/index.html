【王道考研·数据结构】第三章 栈和队列：3.2 队列与双端队列

Austoin2026-03-192026-03-19

引言

在上一节中，我们学习了”后进先出（LIFO）”的特种兵——栈。今天，我们要学习它的兄弟结构：队列（Queue）。

如果说栈是”叠盘子”，那么队列就是我们在食堂打饭时的”排队”。谁先排队，谁就先打到饭。本节不仅涵盖常规队列的顺序和链式实现，还有考研中经常作为拉分项出现的双端队列！

一、队列的基本概念（Queue）

1. 什么是队列？

队列也是一种操作受限的线性表。它只允许在表的一端进行插入，而在表的另一端进行删除。

  • 队头（Front）：允许删除的一端。（也就是排在队伍最前面的人）
  • 队尾（Rear）：允许插入的一端。（也就是刚来排队的人站的位置）
  • 空队列：不含任何元素的空表。

2. 队列的特性：FIFO

队列的逻辑结构属于线性结构，核心特性是：先进先出（First In First Out, FIFO）。

3. 基本操作

  • InitQueue(&Q)：初始化队列。
  • EnQueue(&Q, x)：入队。若队列未满，将 x 加入使其成为新的队尾。
  • DeQueue(&Q, &x)：出队。若队列非空，删除队头元素，并用 x 返回。
  • GetHead(Q, &x)：读队头元素。若队列非空，将队头元素赋给 x（不删除）。
  • QueueEmpty(Q)：判空。

二、队列的顺序实现：循环队列

1. 什么是”假溢出”？

如果只是用普通数组来实现队列，随着元素的不断入队和出队，front 和 rear 指针会一直往后移，最后哪怕数组前面还有空位置，也会提示”队列已满”，这就是所谓的**”假溢出”**。

为了解决这个问题，我们利用取模运算（%），将一维数组在逻辑上首尾相连，形成循环队列！

2. 循环队列的定义

3. 核心难点：如何区分”队空”和”队满”？

在循环队列中，初始时 front = rear = 0。入队时 rear 往后移，出队时 front 往后移。

当队列满了的时候，rear 绕了一圈又追上了 front，此时 rear == front。

这就产生了一个矛盾：队空是 front == rear，队满也是 front == rear，怎么区分？

考研中有以下 三种标准解决方案（必须全部掌握，考题会灵活变通）：

方案一：牺牲一个存储单元（最常用，王道书默认）

让 rear 指向队尾元素的下一个位置。当 rear 的下一个位置是 front 时，就认为队满。

  • 初始化：Q.front = Q.rear = 0;
  • 队空条件：Q.front == Q.rear;
  • 队满条件：(Q.rear + 1) % MaxSize == Q.front;
  • 队列长度：(Q.rear + MaxSize - Q.front) % MaxSize;

方案二：增加一个 size 变量记录长度

不牺牲存储空间，在结构体中增加 int size;。

  • 初始化：Q.front = Q.rear = 0; Q.size = 0;
  • 入队成功：size++;
  • 出队成功：size--;
  • 队空条件：Q.size == 0;
  • 队满条件：Q.size == MaxSize;

方案三：增加一个 tag 变量记录最后一次操作

在结构体中增加 int tag;。每次删除操作成功时，令 tag = 0；每次插入操作成功时，令 tag = 1。

只有”入队”才可能导致队满，只有”出队”才可能导致队空。

  • 队空条件：Q.front == Q.rear && Q.tag == 0;
  • 队满条件：Q.front == Q.rear && Q.tag == 1;

三、队列的链式实现（链队列）

链队列本质上是一个同时带有队头指针和队尾指针的单链表。由于常常要在尾部进行插入，有了队尾指针，入队的时间复杂度就是 $O(1)$。

四、双端队列

1. 什么是双端队列？

双端队列是一种允许在两端都进行插入和删除的线性表。你可以把它理解为”两头通气”的管子。

考研中主要考察它的两个变种以及合法输出序列的判断：

  1. 输入受限的双端队列：只允许从一端插入，但允许从两端删除。
  2. 输出受限的双端队列：允许从两端插入，但只允许从一端删除。

2. 核心考法：判断输出序列合法性

举例：若数据元素输入序列为 1, 2, 3, 4，哪些输出序列是合法的？

解题思路：

  • 栈本质上是一种特殊的双端队列（一端进、同端出）。所以，栈中合法的输出序列，在双端队列中必定也合法！
  • 做这类题目（大多是选择题）时，可以先用”卡特兰数”或者直接判断该序列是不是合法的栈输出序列，如果是，直接打勾。
  • 如果不是合法的栈序列，再代入题目要求的”输入受限”或”输出受限”双端队列中进行模拟，画图推演！

五、经典考点与真题解析

习题 1：循环队列中元素的个数计算

【题目】 已知循环队列存储在一维数组 A[0...n-1] 中，且队列非空时 front 和 rear 分别指向队头元素和队尾元素下一个位置。假设队列满时牺牲一个存储单元，则队列中元素的个数为（ ）。

A. (rear - front + n) % n
B. (rear - front + 1) % n
C. (front - rear + n) % n
D. (rear - front) % n

【解析】 答案选 A。

这是必须背诵并理解的公式！因为是循环队列，rear 有可能在 front 的前面，导致 rear - front 为负数。为了保证算出的是正数的长度，我们需要加上数组的长度 n，然后再对 n 取模。即：(rear + n - front) % n。

习题 2：链队列的特殊边界情况

【题目】 假设用带头结点的单链表表示队列，队头指针 front 指向头结点，队尾指针 rear 指向队尾结点。下列有关出队操作的说法中正确的是（ ）。

A. 仅需修改 front 指针
B. 仅需修改 rear 指针
C. 通常需要修改 front 指针，但在某特殊情况下需要修改 rear 指针
D. 必须同时修改 front 和 rear 指针

【解析】 答案选 C。

回顾我们上面手写的代码：出队操作本质是删除单链表的第一个有效结点，所以通常只需要修改 front->next（即修改头结点的指针）。但是！当队列中只有一个元素时，这个元素出队后队列就空了，此时原本指向这个元素的 rear 指针就会变成悬空指针（野指针）。因此，这种特殊情况下必须把 rear 重新指向头结点（Q.rear = Q.front）。

习题 3：双端队列输出序列判断

【题目】 （2010年统考真题）已知某队列元素的进队序列为 1, 2, 3, 4，并且进队和出队可以穿插进行。若允许从队列的两端出队、仅允许从队列的一端进队（即输入受限的双端队列），则不可能得到的出队序列是（ ）。

A. 4, 1, 3, 2
B. 4, 2, 3, 1
C. 4, 2, 1, 3
D. 3, 4, 2, 1

【解析】 答案选 C。

输入受限：只能从一端进，说明元素在队列内部的相对位置（相对顺序）是固定的，必然是 1-2-3-4 这样排在一起。

假设从右边入队，两端出队。我们要得到第一个输出为 4，说明 1,2,3,4 必须全部先入队。此时队列里的状态从左到右必然是 [1, 2, 3, 4]。

  • 如果要输出 4，可以从右端出队。剩下的队列是 [1, 2, 3]。
  • 接下来，如果要输出 2，无论从左端（弹出1）还是右端（弹出3）出队，都无法直接拿到中间的 2！

因此，在以 4, 2 开头的序列中，4, 2, 1, 3 是无论如何也得不到的。

总结

  1. 循环队列的”模”：只要看到循环队列，指针的加减操作永远要加 % MaxSize，绝不能只写 Q.rear++。

  2. 队满的三种判断方式：考场上一定要先审题，看清楚题目是”牺牲了一个单元”、”加了size变量”还是”加了tag变量”，不同的设定对应不同的满/空公式。

  3. 链队列的出队陷阱：千万别忘了，如果出队的是队列中的最后一个元素，一定要记得处理 rear 指针！

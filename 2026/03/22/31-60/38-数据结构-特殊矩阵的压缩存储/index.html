【王道考研·数据结构】第三章 栈和队列：3.4 特殊矩阵的压缩存储

Austoin2026-03-222026-03-22

引言

在考研数据结构中，矩阵的压缩存储是一个极其重要且容易踩坑的考点。尤其是各种”下标映射”公式，死记硬背很容易在考场上因为”数组下标从0还是从1开始”而全军覆没。

本文将带你彻底搞懂特殊矩阵压缩存储的推导逻辑，告别死记硬背！

一、预备知识：数组的物理存储结构

在进入特殊矩阵之前，我们必须明确数组在内存中是连续存放的。

1. 一维数组

各元素大小相同，物理上连续存放。

  • 设起始地址为 LOC，元素类型大小为 sizeof(ElemType)
  • 寻址公式（下标从0开始）：Address(a[i]) = LOC + i * sizeof(ElemType)

2. 二维数组

二维数组在逻辑上是平面的，但在物理内存中必须”拉平”成一维的。这就产生了两种拉平策略：行优先和列优先。

假设有 $M$ 行 $N$ 列的二维数组 b[M][N]（下标均从0开始）：

行优先存储（C语言默认）：先存第一行，再存第二行…

  • b[i][j] 之前有 $i$ 整行（每行 $N$ 个元素），以及本行前 $j$ 个元素
  • 地址：LOC + (i * N + j) * sizeof(ElemType)

列优先存储：先存第一列，再存第二列…

  • b[i][j] 之前有 $j$ 整列（每列 $M$ 个元素），以及本列前 $i$ 个元素
  • 地址：LOC + (j * M + i) * sizeof(ElemType)

二、对称矩阵的压缩存储

压缩存储的理念： 找出矩阵中规律性重复的元素，只存一份；常数元素只存一个。

1. 什么是对称矩阵？

对于一个 $n$ 阶方阵，若任意元素满足 $a_{i,j} = a_{j,i}$，则为对称矩阵。

2. 压缩策略

普通存储需要 $n \times n$ 的一维数组。因为上下三角对称，我们只需存储主对角线 + 下三角区（或上三角区）。

需要的数组大小为：$1 + 2 + 3 + … + n = \frac{n(n+1)}{2}$。

我们定义一维数组 B[n(n+1)/2]。

3. 下标映射公式推导

大前提（极易错点）： 题目中矩阵的行、列号通常从 1 开始，而存入的一维数组下标从 0 开始！请务必仔细审题！

按【行优先】将【下三角区】存入一维数组 B[k]：

设需要求 $a_{i,j}$ ($i \ge j$，即下三角区元素) 对应在一维数组中的下标 $k$：

  1. 它前面有多少个完整的行？ 有 $i-1$ 行

    • 前 $i-1$ 行的元素总数为：$1 + 2 + … + (i-1) = \frac{i(i-1)}{2}$ 个

  2. 它是当前第 $i$ 行的第几个元素？ 第 $j$ 个

  3. 计算下标 $k$： 因为数组下标从0开始，所以它前面的元素总数就是它的下标！

$$k = \frac{i(i-1)}{2} + j - 1$$

若访问的是上三角区元素 ($i < j$)，怎么办？

利用对称性 $a_{i,j} = a_{j,i}$，直接将公式里的 $i$ 和 $j$ 互换即可：

$$k = \frac{j(j-1)}{2} + i - 1$$

三、三角矩阵的压缩存储

1. 什么是三角矩阵？

  • 下三角矩阵： 主对角线及下三角区有各不相同的元素，上三角区全为同一个常量 $c$
  • 上三角矩阵： 主对角线及上三角区有各不相同的元素，下三角区全为同一个常量 $c$

2. 压缩策略

和对称矩阵完全一样，只是在数组的最后多加一个位置用来存储那个常量 $c$。

需要的数组大小为：$\frac{n(n+1)}{2} + 1$。

3. 下标映射（以下三角矩阵、行优先为例）

  • 当下三角区及主对角线 ($i \ge j$)：

$$k = \frac{i(i-1)}{2} + j - 1$$

  • 当上三角区 ($i < j$)：

全都映射到最后一个元素：$k = \frac{n(n+1)}{2}$

4. 代码实现

四、三对角矩阵的压缩存储

1. 什么是三对角矩阵？

当 $|i - j| > 1$ 时，有 $a_{i,j} = 0$。也就是说，所有的非零元素都集中在主对角线及其上下相邻的两条斜线上，像一条带子。

2. 压缩策略

只存储带状部分的非零元素。按行优先存储存入一维数组中。

  • 第 1 行和第 $n$ 行各有 2 个元素
  • 第 2 到第 $n-1$ 行各有 3 个元素

总元素个数：$3n - 2$。

3. 下标映射公式

已知矩阵下标 $i, j$，求一维数组下标 $k$：

要求 $a_{i,j}$ 前面有多少个元素：

  1. 前 $i-1$ 行的总元素数： 第一行2个，第2到$i-1$行各3个。公式化简后为：$3(i-1) - 1$ 个
  2. 当前行它前面有几个： $j - i + 1$ 个
  3. 相加得到 $k$ (数组从0开始)：

$$k = 2i + j - 3$$

已知一维数组下标 $k$，求矩阵下标 $i, j$：

  • 推导行号 $i$：$i = \lfloor(k+1)/3 + 1\rfloor$ （向下取整） 或 $i = \lceil(k+2)/3\rceil$ （向上取整）
  • 推导列号 $j$：代入上方的公式，得到 $j = k - 2i + 3$

五、稀疏矩阵的压缩存储

1. 什么是稀疏矩阵？

非零元素个数远远少于矩阵元素的总数。

2. 压缩策略（失去随机访问特性）

由于零元素太多，我们只记录非零元素的位置和值。

方法一：顺序存储 —— 三元组表

只记录非零元素的：<行号, 列号, 值>。

方法二：链式存储 —— 十字链表法

当稀疏矩阵经常需要进行插入、删除运算时，三元组表移动元素的代价太大，此时使用十字链表。

每一个非零元素作为一个结点：

  • 数据域： row, col, value (行、列、值)
  • 指针域：
    • right: 指向向右的下一个同行的非零元素
    • down: 指向向下的下一个同列的非零元素

六、做题防坑指南

  1. 看清起点： 矩阵的行、列下标是从 0 开始还是从 1 开始？一维数组的下标是从 0 开始还是从 1 开始？（考试最喜欢在这里挖坑！）

  2. 理清方向： 是行优先存储，还是列优先存储？

  3. 上或下？ 是存上三角还是下三角？

  4. 遇事不决，代入数据： 考场上如果忘了公式，千万不要凭空想！直接在草稿纸上画一个 $3 \times 3$ 或 $4 \times 4$ 的矩阵，标上序号 $(1,1), (1,2)…$，自己动手把前几个元素的下标写出来，找等差数列的规律！

七、经典考点与课后习题解析

习题 1：对称矩阵列优先存储

【题目】 将一个 $10 \times 10$ 的对称矩阵 $M$ 的上三角部分的元素 $m_{i,j}$ ($1 \le i \le j \le 10$) 按列优先存入C语言的一维数组 $N$ 中，元素 $m_{7,2}$ 在 $N$ 中的下标是？

【解析】

  1. 审题防坑： 矩阵行列从 1 开始，存入的是C语言数组（下标从0开始）。对称矩阵，虽然问的是 $m_{7,2}$，但由于只存了上三角，我们需要利用对称性将其转换为上三角元素 $m_{2,7}$

  2. 明确策略： 存上三角，列优先。意思是：先存第1列，再存第2列…

  3. 计算过程： 我们要求 $m_{2,7}$ 前面有多少个元素

    • 它在第 7 列，那么前面的 1 到 6 列都已经全部存满
    • 第 1 列存 1 个，第 2 列存 2 个 … 第 6 列存 6 个
    • 前 6 列总数：$1+2+3+4+5+6 = 21$ 个
    • 在第 7 列中，$m_{2,7}$ 是第 2 个元素（它前面还有1个 $m_{1,7}$）
    • 所以它前面共有 $21 + 1 = 22$ 个元素

  4. 得出答案： 因为数组下标从 0 开始，前面有 22 个元素，所以它的下标就是 22

习题 2：三对角矩阵公式反推

【题目】 设有一个按行优先存储在连续内存中的三对角矩阵，已知矩阵的第一行第一列元素位于数组下标 k=0 处。请问数组下标 k=14 对应矩阵的行号 $i$ 和列号 $j$ 分别是多少？（矩阵行列从 1 开始计算）

【解析】

  1. 明确公式： 题目符合标准的三对角矩阵定义。我们已知数组下标 $k=14$

  2. 求行号 $i$： 根据公式 $i = \lfloor(k+1)/3 + 1\rfloor$ 或 $i = \lceil(k+2)/3\rceil$

    • 代入 $k=14$：$i = \lfloor 15/3 + 1 \rfloor = \lfloor 5 + 1 \rfloor = 6$
    • 所以该元素在第 6 行

  3. 求列号 $j$： 我们知道由 $(i,j)$ 变 $k$ 的公式是 $k = 2i + j - 3$

    • 现在已知 $k=14, i=6$，代入公式：
    • $14 = 2 \times 6 + j - 3$
    • $14 = 12 + j - 3 \Rightarrow j = 5$

  4. 得出答案： 对应矩阵的行号 $i=6$，列号 $j=5$，即 $a_{6,5}$

习题 3：稀疏矩阵辨析

【题目】 关于稀疏矩阵，以下说法正确的是（ ）

A. 稀疏矩阵由于零元素多，所以应该用一维数组顺序存储
B. 将稀疏矩阵转化为三元组表存储后，依然保留了随机存取特性
C. 十字链表法可以有效解决三元组表在插入、删除非零元素时需要大量移动数据的问题
D. 特殊矩阵（如对称矩阵）和稀疏矩阵的压缩存储方法是相同的

【解析】

  • A 错误：稀疏矩阵通常使用三元组或十字链表，而不是简单的一维数组顺序存储（一维数组是普通/特殊矩阵的存法）

  • B 错误：转化为三元组后，要查找某个特定的 $(i,j)$ 必须从头遍历三元组，失去了随机存取特性（这也是为什么时间复杂度可能变高的原因）

  • C 正确： 链表天生适合插入和删除，十字链表完美解决了这个问题

  • D 错误：特殊矩阵（如对称矩阵）是用一维数组映射；稀疏矩阵是用三元组/十字链表，两者完全不同

答案选 C

总结

  1. 对称矩阵：利用对称性，只存储下三角或上三角，公式 $k = \frac{i(i-1)}{2} + j - 1$

  2. 三角矩阵：与对称矩阵类似，但需要额外存储常量 $c$

  3. 三对角矩阵：只存储带状区域，公式 $k = 2i + j - 3$

  4. 稀疏矩阵：使用三元组表或十字链表，失去随机访问特性

矩阵的压缩存储是纸老虎，切记画图推导 > 死记公式。学会使用等差数列求和公式，搞清起始下标，你就能在考场上秒杀这类题目！
